## Что такое tokmax (ведж)

Одна команда `npx tokmax` сканирует локальные логи **Codex + Claude Code** и
показывает **API-equivalent $** — сколько твой usage стоил бы по ценам API.
Вирусный артефакт: люди скриншотят своё число и хвастаются. Лидерборд =
соревновательность. Наружу уходят только агрегаты-числа (не код, не ключи).

## Почему расходится (вирусные механики)

- **Флекс/статус** — большое число = понт, им делятся.
- **Шок-цифра** — «я нажёг на $2870?!» = шерится само.
- **Соревнование** — лидерборд, «кто больше».
- **Ноль трения** — одна команда, без установки.

## Игра Б (почему это для тебя открыто)

Продукт-лед лонч на discovery-платформах. Паспорт / кредлы / акцент не значат
ничего — тула говорит сама. Не нужен ни VC, ни личный бренд. Образец пути —
Pieter Levels (levelsio): не-native, без кредлов, без VC, крошечные тулзы → большая
западная аудитория. Логика РУ/ENG-сплита: бренд → РУ (твоя сила), тулзы → глобал
(паспорт-агностик).

## Аудитория и где она

Power-юзеры Codex + Claude Code (кому их token-burn либо в шок, либо в гордость).
Тусуются: **Twitter/X** (AI-dev), **r/ClaudeAI**, **r/ChatGPTCoding**,
**Hacker News**, **Product Hunt**.

## План лонча (последовательность)

0. **Pre-launch gates** (см. ниже — без них не запускаемся широко).
1. Двуязычная страница + GitHub-полировка (README, лого, 2-3 живых профиля-семпла, GIF).
2. **Show HN** — "Show HN: tokmax – what your Claude Code / Codex usage would cost at API prices".
3. **Reddit** — r/ClaudeAI, r/ChatGPTCoding (возможно r/LocalLLaMA, r/artificial).
4. **Product Hunt** — запланированный, старт 00:01 PT, полный листинг.
5. **Twitter-тред** — build-in-public, флекс + лидерборд.
6. Сбор фидбэка (агент пылесосит реплаи → дайджест) → итерации.

## ⚠️ Launch-readiness gates (ОБЯЗАТЕЛЬНО до широкого лонча)

Вирусный трафик на текущей связке = риск. До PH/HN закрыть:

1. **Prod Convex** — сейчас dev `chatty-boar-479`. Под нагрузку нужен прод-деплой.
2. **3 P1 безопасности** (из аудита) — это публичный анонимный write-endpoint:
   (a) XFF rate-limit bypass; (b) глобальный дневной потолок = DoS-вектор;
   (c) **leaderboard poisoning** (сервер верит присланным числам → борд затроллят
   в первый день фейковыми $-рекордами). До публичного лонча — закрыть.
3. **Нагрузка/лимиты** — убедиться, что endpoint держит пик.
4. **Trust-месседж** — кристально ясно на странице и в постах: «читает только
   агрегаты-числа, не код и не ключи, open-source — проверь сам». На западе доверие
   к тулзе, которая лезет в твои логи, — вопрос №1.

## Что защищает РУ-трек

- Отдельный репо + отдельные каналы (РУ-бренд не участвует и не упоминается).
- Продукт-лед: твоя персона/лицо не нужны.
- Спринт (2-3 дня всплеска), не параллельный трек навсегда.
- ~90% работы — на агенте; владелец только публикует со своих аккаунтов +
  лёгкое присутствие в день лонча.

## Доля владельца (минимум)

Аккаунты твои (PH / HN / Reddit / Twitter), публикуешь сам по готовым черновикам
(платформы не любят ботов, и агент не постит за тебя без твоего ок). День лонча:
подтвердить + поотвечать на коменты (черновики ответов готовит агент).

## Метрики успеха

npm downloads, GitHub stars, опубликованные профили, записи лидерборда,
твиттер-импрешены, качество фидбэка. Зашло → даблдаун + думаем про монетизацию.

## Монетизация (потом, не сейчас)

Сначала бесплатно → охват. Позже — Merchant-of-Record (Paddle / Lemon Squeezy /
Polar) + зарубежное юрлицо (Аргентина / Эстония / …). Лонч на это НЕ завязываем.
