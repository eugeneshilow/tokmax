# Launch Runbook

Before launch:

- Verify the production web app and Convex deployment.
- Run a clean-machine publish with the latest CLI.
- Confirm profile pages and leaderboards render fresh data.
- Confirm no regional references remain in public pages.
- Confirm README, npm metadata, and web copy describe the same privacy boundary.

Launch day:

- Post from the owner's X account.
- Share the GitHub repository.
- Monitor publish errors and leaderboard abuse signals.
- Keep replies focused on privacy, methodology, and open-source verification.

After launch:

- Review publish volume and error logs.
- Recompute suspicious profiles if needed.
- Patch copy or methodology docs if users find ambiguity.
