# Marketing

Launch materials live in `launch/`. The narrative spine everywhere is
**tokenmaxxing**: the flex (verified burn leaderboard) + the optimization
(PROFIT/×, second-subscription stack math). Keep all public copy
English-only; public links point to the product, the GitHub repository, or
the owner's X profile. No regional brand references.

Launch assets (in posting order):

- [launch/x-launch.md](launch/x-launch.md) — X: launch thread, daily cadence,
  standalone posts, viral mechanics (runs DURING the event window).
- [launch/channels.md](launch/channels.md) — master multi-channel plan:
  sequencing, communities, newsletters, blog, GitHub, Codex delegation brief
  (runs AFTER the event, with its data).
- [launch/product-hunt.md](launch/product-hunt.md) — PH listing, maker
  comment, gallery spec, launch-day playbook.
- [launch/show-hn.md](launch/show-hn.md) — Show HN titles, text,
  comment-defense FAQ.
- [launch/reddit.md](launch/reddit.md) — per-subreddit texts + rules of
  engagement.
- [launch/strategy.md](launch/strategy.md) — positioning and trust messaging.
- [launch/runbook.md](launch/runbook.md) — release readiness gates.
